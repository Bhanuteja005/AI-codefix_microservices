Hardcoded Credentials (CWE-798) - Security Guidelines

**Description:**
Hardcoded credentials involve embedding passwords, API keys, or secrets directly in source code. This creates security risks as credentials can be exposed through version control, decompilation, or unauthorized access.

**Secure Remediation:**

1. **Use Environment Variables**
   - Store secrets in environment variables
   - Example (Python): `password = os.environ.get('DB_PASSWORD')`
   - Example (Java): `String password = System.getenv("DB_PASSWORD")`

2. **Use Secret Management Systems**
   - AWS Secrets Manager
   - Azure Key Vault
   - HashiCorp Vault
   - Google Secret Manager

3. **Configuration Files (Outside Version Control)**
   - Use .env files with .gitignore
   - Store config files outside the repository
   - Never commit secrets to version control

4. **Use Secure Credential Injection**
   - CI/CD pipeline secret injection
   - Container secret mounting
   - Runtime secret retrieval

**Example Fix:**

Vulnerable:
```python
db_password = "MySecretPassword123!"
connection = db.connect(host="localhost", password=db_password)
```

Secure:
```python
import os
db_password = os.environ.get('DB_PASSWORD')
if not db_password:
    raise ValueError("DB_PASSWORD environment variable not set")
connection = db.connect(host="localhost", password=db_password)
```
